What is Stratum?

Stratum inverts the conventional relationship between CPUs and ASICs. While SHA-256 strips a processor to its core arithmetic operations (creating ~100,000× efficiency gaps), Stratum makes CPU overhead—cache hierarchies, speculative execution, branch prediction—the foundation of the algorithm itself.

An ASIC attempting to replicate Stratum's behavior would, by definition, become a CPU. This creates structural parity, making CPU mining economically viable once again.

Core insight: "The right frame is not resistance — it is inversion. Design a hash where the expensive overhead of a CPU is the computation, not a tax on it."

How It Works

Stratum solves four critical engineering problems with targeted technical innovations:

1. Verification Asymmetry — ZK Proofs of Cache Execution

Miners generate a STARK proof alongside each hash, proving the cache access sequence was executed correctly. Full nodes verify in O(log n) time (~50ms) without re-running the computation. No trusted setup required.

2. Determinism — Software Cache Model

Rather than relying on variable physical cache behavior, Stratum defines a canonical software cache model: L1 (256 KB), L2 (1 MB), L3 (12 MB) with LRU eviction. Guarantees identical output across all compliant hardware.

3. Portability — ISA Profile System

Stratum specifies four ISA profiles, each optimized for different architectures. The network accepts any profile that produces valid hashes against the canonical model.

4. ASIC Resistance — Speculative Trace Commitment (v1.5)

Miners log execution traces through a bounded 32-instruction speculative window. Verifiers replay branch points using a deterministic branch prediction emulator. Requires actual speculative execution—the majority cost of building a modern CPU core.
